I. Context¶

  • Initial goal : categorize pixel in cloud or not cloud pixels.
  • How: throw 13 different methods (10 based on difference between the image and non cloud ground - 3 ready to use decision trees).
  • Dataset : public dataset (repo) answering the question if a pixel is a cloud or not.
  • Goal of the report: build a robust cloud detection model based and our set of methods

II. Current situation¶

  • Methods processing: the main buggs present in our methods has been fixed

    • Not sqare image : all the image haven't the same shape (solved by adding a common area criteria)
    • Duplicated images : some images have been sampled at the same date but prodided at different dates. As result, there are duplicated image with different ids (solved by filtering the sample date and distribution date)
    • Full mask bands : some extra bands are always empty (QA10 - QA20) (solved by ignoring those problematic bands)

  • After processing all those images, the methods still fail on some pixels. That lead use to remove :

    • Training set : 3036 pixels from the 200 000 as input (1.5%)
    • Evaluation dataset: 924 pixels from the 60 000 as input (1.5%)

    => The final valid datset in composed by 196964 samples and the evaluation dataset by 59076 pixels.

III. Methods¶

III.1. Random Forest¶

I used the randomForestClassifier method from the sklearn package (doc). A lot of options are available. One really import option is the n_estimators parameter. It's the number of samples used for building the randomForest model. Obviously, the bigger it is, the better the results are. However, using a lot of estimators is ressource consuming.

The following figure has 2 plots:

  • Plot 1: accuracy over the number of estimators
